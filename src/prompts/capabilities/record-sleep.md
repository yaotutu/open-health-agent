# 睡眠记录

## 工具
`record_sleep`

## 参数
| 参数 | 类型 | 说明 |
|------|------|------|
| duration | number | 睡眠时长 (分钟) |
| quality | number | 睡眠质量 (1-5) |
| bedTime | number | 入睡时间 (毫秒时间戳) |
| wakeTime | number | 醒来时间 (毫秒时间戳) |
| deepSleep | number | 深睡时长 (分钟) |
| note | string | 备注 |

## 使用时机
- 用户提到睡眠（如"昨晚睡得不好"、"今天睡了个懒觉"）
- 用户描述睡眠质量或时长
- 用户提到睡觉时间或起床时间（如"两点才睡"、"八点起床"）
- 用户发送睡眠相关的截图或数据

## 注意事项
- duration 是核心信息，优先获取
- 用户说"睡了8个小时"时，转换为分钟（480）
- quality 如果用户没说，可以根据描述推断（"睡得很好"→4-5，"一般"→3，"没睡好"→1-2）

## 时间提取规则（重要）

当用户消息中包含任何与睡觉/起床时间相关的描述时，**必须主动提取**并转换为毫秒时间戳填入 bedTime 和 wakeTime。

### 提取原则
- 用户不一定直接说"入睡时间"或"醒来时间"，而是用日常用语描述（如"两点才睡"、"八点起"、"凌晨三点才睡着"）
- 即使用户没有明确要求记录，只要消息中包含了这些信息，就应该提取
- 如果只提到睡觉时间或只提到起床时间，就只填一个字段

### 时间转换方法
1. 基于当前日期计算：将用户说的时间点转换为当天（或前一天，如果是深夜）对应的时间戳
2. 如果用户说"两点睡觉"，通常指今天凌晨2:00（即今天 02:00 的时间戳）
3. 如果用户说"昨晚两点睡觉"，指昨天凌晨2:00
4. 如果用户说"八点起床"，指今天早上 08:00

### 示例
| 用户说法 | bedTime | wakeTime |
|---------|---------|----------|
| "两点睡觉，八点起床" | 今天 02:00 的时间戳 | 今天 08:00 的时间戳 |
| "昨晚12点睡的" | 昨天 00:00 的时间戳 | 不填 |
| "凌晨三点才睡着，早上七点半就醒了" | 今天 03:00 的时间戳 | 今天 07:30 的时间戳 |
| "十一点睡觉六点半起" | 昨天 23:00 的时间戳 | 今天 06:30 的时间戳 |

### 时间戳计算
根据系统注入的"当前时间"计算：`new Date(年, 月-1, 日, 时, 分).getTime()`
例如当前时间是 2026年3月28日周六，用户说"两点睡觉，八点起床"：
- bedTime = new Date(2026, 2, 28, 2, 0).getTime() = 今天凌晨2:00的时间戳
- wakeTime = new Date(2026, 2, 28, 8, 0).getTime() = 今天早上8:00的时间戳

### 多日场景
当用户说"连续两天"或"最近两天"时，需要为每一天分别调用 record_sleep：
- 先根据当前时间计算出"昨天"和"前天"的具体日期
- 然后为每一天分别生成带对应日期的 bedTime/wakeTime 并调用工具
- 例如用户说"连续两天都是两点睡觉，八点起床"，当前是3月28日：
  - 第1次调用：bedTime=3月28日02:00, wakeTime=3月28日08:00
  - 第2次调用：bedTime=3月27日02:00, wakeTime=3月27日08:00
