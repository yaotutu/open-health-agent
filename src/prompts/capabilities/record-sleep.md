# 睡眠记录

## 工具
`record_sleep`

## 参数
| 参数 | 类型 | 说明 |
|------|------|------|
| duration | number | 睡眠时长 (分钟) |
| quality | number | 睡眠质量 (1-5) |
| bedTime | string | 入睡时间，格式 "YYYY-MM-DD HH:mm" |
| wakeTime | string | 醒来时间，格式 "YYYY-MM-DD HH:mm" |
| deepSleep | number | 深睡时长 (分钟) |
| note | string | 备注 |

## 使用时机
- 用户提到睡眠（如"昨晚睡得不好"、"今天睡了个懒觉"）
- 用户描述睡眠质量或时长
- 用户提到睡觉时间或起床时间（如"两点才睡"、"八点起床"）
- 用户发送睡眠相关的截图或数据

## 注意事项
- duration 是核心信息，优先获取
- 用户说"睡了8个小时"时，转换为分钟（480）
- quality 如果用户没说，可以根据描述推断（"睡得很好"→4-5，"一般"→3，"没睡好"→1-2）

## 去重规则（重要）
**创建睡眠记录前，必须先查询是否已有同一天的记录，避免重复。**

1. 调用 `query_sleep_records` 查询最近几天的睡眠记录
2. 根据已有记录的 bedTime 或 timestamp 判断目标日期是否已有记录
3. 如果目标日期已有记录，**不要重复创建**，跳过该日期
4. 只为没有记录的日期创建新记录

示例：用户说"连续两天两点睡八点起"，查询后发现今天已有记录，则只为昨天创建。如果两天都已有记录，则不调用 record_sleep，直接告诉用户已记录过。

## 时间提取规则（重要）

当用户消息中包含任何与睡觉/起床时间相关的描述时，**必须主动提取**并以 "YYYY-MM-DD HH:mm" 格式填入 bedTime 和 wakeTime。不要计算时间戳，只需填入日期时间字符串，系统会自动转换。

### 提取原则
- 用户不一定直接说"入睡时间"或"醒来时间"，而是用日常用语描述（如"两点才睡"、"八点起"、"凌晨三点才睡着"）
- 即使用户没有明确要求记录，只要消息中包含了这些信息，就应该提取
- 如果只提到睡觉时间或只提到起床时间，就只填一个字段

### 时间确定方法
根据系统注入的"当前时间"确定具体日期：
1. "两点睡觉" → 通常指今天凌晨2:00，即 bedTime = "当天日期 02:00"
2. "昨晚两点睡觉" → 昨天凌晨2:00
3. "八点起床" → 今天早上 08:00
4. "十一点睡" → 昨天晚上 23:00

### 示例（假设当前 2026-03-28）
| 用户说法 | bedTime | wakeTime |
|---------|---------|----------|
| "两点睡觉，八点起床" | "2026-03-28 02:00" | "2026-03-28 08:00" |
| "昨晚12点睡的" | "2026-03-27 00:00" | 不填 |
| "凌晨三点才睡着，七点半醒了" | "2026-03-28 03:00" | "2026-03-28 07:30" |
| "十一点睡觉六点半起" | "2026-03-27 23:00" | "2026-03-28 06:30" |

### 多日场景
当用户说"连续两天"时，根据当前时间计算出具体日期，为每一天分别调用 record_sleep：
- 例如用户说"连续两天都是两点睡八点起"，当前是3月28日：
  - 第1次调用：bedTime="2026-03-28 02:00", wakeTime="2026-03-28 08:00"
  - 第2次调用：bedTime="2026-03-27 02:00", wakeTime="2026-03-27 08:00"
